# Contributing to Claude5*

Thank you for your interest in contributing to Claude5*! This document provides guidelines and instructions for contributing.

## üéØ How to Contribute

### Reporting Issues

- Use the [issue template](.github/ISSUE_TEMPLATE.md) when reporting bugs or requesting features
- Include as much detail as possible: steps to reproduce, expected behavior, environment details
- Search existing issues before creating new ones

### Submitting Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit your changes (`git commit -m 'feat: add amazing feature'`)
5. Push to the branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

### Code Style

- Keep documentation clear and concise
- Use conventional commit messages: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`
- Update relevant documentation when making changes
- Test your changes before submitting

## üìö Documentation Updates

When updating documentation:

- Keep `.claude/CLAUDE.md` as the single source of truth for project configuration
- Update `README.md` for user-facing changes
- Update `.docs/FIVE-ECOSYSTEMS-SETUP.md` for ecosystem-related changes
- Maintain consistency across all documentation files

## üîß Development Setup

1. Clone your fork: `git clone https://github.com/YOUR_USERNAME/claude5-starter-kit.git`
2. Navigate to the project: `cd claude5-starter-kit`
3. Install the 5 required ecosystems (see [FIVE-ECOSYSTEMS-SETUP.md](.docs/FIVE-ECOSYSTEMS-SETUP.md))
4. Make your changes
5. Test thoroughly

## ‚úÖ PR Review Process

All PRs go through review:

1. Automated checks (if configured)
2. Manual review by maintainers
3. Feedback and requested changes
4. Approval and merge

## üéì Project Philosophy

Claude5* is built on these principles:

1. **Test-Driven Development** - Tests first, always
2. **Systematic Process** - Methodology over guessing
3. **Simplicity** - Complexity reduction
4. **Evidence** - Verify before claiming success
5. **Efficiency** - Work smart, use voice, optimize tokens
